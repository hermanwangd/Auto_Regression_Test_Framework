# v0.2.1 Usage Kit Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish Framework `0.2.1` with a curated usage kit release artifact while keeping the public contract/schema version at `v0.2`.

**Architecture:** Treat `0.2.1` as a release-packaging hardening patch, not a contract redesign. The release jar remains the executable artifact; the usage kit is a verified companion zip containing user guide, Provider Contract catalog, schemas, and runnable checked-in samples.

**Tech Stack:** Spring Boot 3.x, Java 17+, Maven Wrapper, Bash release scripts, GitHub Actions release workflow, YAML/JSON contract artifacts.

### Task 3: Build Usage Kit Script

**Files:**
- Create: `scripts/release/build-usage-kit.sh`

- [ ] **Step 1: Add the build script**

Create an executable Bash script that:

```bash
#!/usr/bin/env bash
set -euo pipefail

VERSION="${1:-}"
if [[ -z "$VERSION" ]]; then
  echo "Usage: $0 <version>" >&2
  exit 2
fi

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
cd "$ROOT_DIR"

scripts/release/verify-release-version.sh "$VERSION" >/dev/null

OUT="target/spec-driven-auto-regression-${VERSION}-usage-kit.zip"
mkdir -p target
WORK_DIR="$(mktemp -d target/usage-kit.XXXXXX)"
trap 'rm -rf "$WORK_DIR"' EXIT

KIT_ROOT="$WORK_DIR/usage-kit"
mkdir -p "$KIT_ROOT"
```

Then copy only approved inputs:

```bash
mkdir -p "$KIT_ROOT/docs/09-operations" \
  "$KIT_ROOT/docs/08-release/framework" \
  "$KIT_ROOT/docs/02-architecture/contracts" \
  "$KIT_ROOT/samples"

cp -R docs/09-operations/. "$KIT_ROOT/docs/09-operations/"
cp -R docs/08-release/framework/. "$KIT_ROOT/docs/08-release/framework/"
cp -R docs/02-architecture/contracts/. "$KIT_ROOT/docs/02-architecture/contracts/"
cp -R schemas "$KIT_ROOT/schemas"
cp -R samples/golden_e2e "$KIT_ROOT/samples/golden_e2e"
cp -R samples/contract_baseline "$KIT_ROOT/samples/contract_baseline"
cp -R samples/provider_capability "$KIT_ROOT/samples/provider_capability"
cp -R samples/evidence_hardening "$KIT_ROOT/samples/evidence_hardening"
cp CHANGELOG.md "$KIT_ROOT/CHANGELOG.md"
```

Expected: `target/spec-driven-auto-regression-0.2.1-usage-kit.zip` contains a single root folder named `usage-kit/`.
