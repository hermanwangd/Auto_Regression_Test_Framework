# 12. PR Review / Evidence Checklist

## 12.1 Spec Review

- [ ] Feature ID is clear.
- [ ] Acceptance criteria are clear.
- [ ] Input/output are defined.
- [ ] Validation rules are defined.
- [ ] Architecture touchpoints are clear.
- [ ] Regression required flag is defined.
- [ ] Priority is defined.

## 12.2 Test Artifact Review

- [ ] Regression matrix updated.
- [ ] Test case YAML follows schema.
- [ ] Data binding is valid.
- [ ] Fixture and cleanup are safe.
- [ ] Expected result is reasonable.
- [ ] P1 test has approval.
- [ ] No secret or sensitive data is committed.

## 12.3 Framework Review

- [ ] Adapter usage is correct.
- [ ] Assertion type is appropriate.
- [ ] Oracle source is valid.
- [ ] Report contains evidence.
- [ ] Failure classification is available.
- [ ] Release gate rule is not bypassed.
